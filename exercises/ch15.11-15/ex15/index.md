## Slack や O365 の API は機密情報をヘッダーで指定する。パスやクエリでないのは何故かセキュリティの観点で記載しなさい。

### パスやクエリでないのは何故か

- 機密情報がURLに含まれてしまうため、サーバーログやブラウザの履歴などに記録される可能性があり、セキュリティリスクとなる
- 他のウェブサイトにリンクされた場合、機密情報がリファラ情報(どのURLからやってきたのか情報)に含まれる可能性がある
- ヘッダーであれば、HTTPSを使用して通信する場合、リクエストヘッダー内のデータは暗号化され、盗聴者からの保護が得られるし、URLやリファラ情報に含まれることもない。
